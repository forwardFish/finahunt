# Epic 2.3 Result Warehouse and Fermentation Engine

## Epic 目标

把 Sprint 2 前半段已经得到的候选事件、题材候选和结构化卡片，进一步沉淀成：

- 可复用、可追溯、可回放的结构化结果仓库
- 可按时间窗口计算的题材热度快照
- 可直接被用户消费的发酵题材候选结果流

## Epic 边界

### 做什么

- 持久化 raw / normalized / canonical / theme / result-card 结果
- 生成题材热度与加速度快照
- 产出“正在发酵”候选流

### 不做什么

- 不做复杂页面
- 不做通知推送
- 不做历史复盘推荐

## Story 清单

| Story | 名称 | 一句话说明 |
| :--- | :--- | :--- |
| S2-007 | 结构化结果仓库 | 把中间结果和最终结果真正落盘成可追溯仓库 |
| S2-008 | 题材热度快照与发酵评分 | 按时间窗口生成题材热度、加速度和发酵评分 |
| S2-009 | 发酵题材候选结果流 | 生成可以直接给用户消费的“发酵题材候选”结果流 |
