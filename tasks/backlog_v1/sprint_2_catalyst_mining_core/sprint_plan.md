# Sprint 2 规划

## Sprint 名称

潜在题材催化结构化挖掘核心

## Sprint 目标

把 Sprint 1 已经筛出来的有效资讯、标准化事件和基础题材标签，进一步加工成：

- 可持续落盘的结构化结果仓库
- 可解释的题材热度快照
- 可排序的“正在发酵题材”候选结果

让用户不只是看到单条资讯，而是能一眼判断：

**哪些题材正在形成新催化，哪些只是噪声。**

## 优先级说明

这是当前 backlog 里第一优先级的业务 Sprint。

## 业务边界

### 做什么

- 提取候选催化句
- 识别催化类型
- 判断催化强度和时效性
- 聚合潜在题材候选
- 建立题材到个股/板块的客观映射
- 产出结构化催化结果卡片
- 建立结构化结果仓库
- 生成题材热度快照和发酵评分
- 产出发酵题材候选结果流

### 不做什么

- 不做复杂前端页面
- 不做通知推送
- 不做历史相似案例推荐
- 不做收益预测或交易建议

## Epic 总览

| Epic | Story 数 | 核心职责 |
| :--- | :--- | :--- |
| Epic 2.1 Catalyst Signal Extraction | 3 | 从内容里提取、分类、分级催化信号 |
| Epic 2.2 Theme Candidate Structuring | 3 | 把催化信号组织成题材候选和结构化结果卡片 |
| Epic 2.3 Result Warehouse and Fermentation Engine | 3 | 沉淀结构化结果仓库，并判断哪些题材正在发酵 |
