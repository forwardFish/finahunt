# Sprint 1 规划

## Sprint 名称

事件认知闭环 MVP

## Sprint 目标

把资讯接入、清洗去重、事件抽取、题材认知、用户排序和日终复盘串成最小可用业务闭环。

## 业务边界

### 做什么

- 跑通资讯获取到标准化资讯
- 跑通事件抽取和事件归一
- 跑通题材识别、催化识别、个股/板块关联
- 跑通用户相关度排序
- 跑通今日重点、观察池事件、日终复盘三个输出

### 不做什么

- 不做社区和内容分享
- 不做复杂通知体系
- 不做荐股、预测或交易功能

## Epic 总览

| Epic | Story 数 | 核心职责 |
| :--- | :--- | :--- |
| Epic 1.1 Ingest and Event | 4 | 获取资讯、清洗去重、抽取事件、事件归一 |
| Epic 1.2 Theme and Linkage | 3 | 题材识别、催化分级、个股/板块关联 |
| Epic 1.3 User Outputs | 2 | 用户相关度排序和日终复盘输出 |
