## 1. 这次 MVP 最重要的目标

Finahunt 当前阶段不是做“更多资讯”，而是做“更高价值的资讯理解结果”。

这版 MVP 的第一优先目标是：

**潜在题材催化的结构化挖掘结果**

也就是把分散资讯压缩成一类用户真正关心的认知结果：

- 哪些事件可能正在形成新的题材催化
- 催化属于什么类型
- 催化强度和时效性怎么样
- 对应哪些板块、个股、产业链环节
- 有哪些来源和证据支撑

如果这件事做不出来，其他页面和排序都只是包装层。

## 2. MVP 一句话定义

面向 A 股高频复盘和题材研究用户，围绕用户的观察池和关注题材，完成：

`资讯接入 -> 清洗去重 -> 事件抽取 -> 潜在催化识别 -> 题材结构化 -> 关联标的 -> 用户排序 -> 日终复盘`

的最小业务闭环。

## 3. MVP 成功标准

MVP 只验证三件事：

1. 系统能否稳定产出“潜在题材催化的结构化结果”。
2. 用户能否用这个结果更快定位当日值得研究的方向。
3. 最终输出是否合规、可追溯、可人工复核。

## 4. 功能优先级

### P0 第一优先主线

1. 合规资讯源接入
2. 资讯清洗与去重
3. 事件抽取与归一
4. 候选催化句抽取
5. 催化类型识别
6. 催化强度与时效性判断
7. 潜在题材聚合
8. 个股 / 板块 / 产业链客观关联
9. 用户相关度排序
10. 今日重点事件 / 观察池事件 / 日终复盘输出
11. 合规治理与人工校正

### P1 第二优先主线

1. 异动预警中心
2. 历史相似案例基础版
3. 研究归档
4. 个股 / 题材详情增强
5. 轻量通知

### P2 暂缓

1. 社区和 UGC
2. 复杂个性化规则引擎
3. 商业化系统
4. App / 小程序
5. 荐股、预测、自动交易

## 5. MVP 不做什么

这次范围明确不做：

- 不做实时交易系统
- 不做荐股和买卖建议
- 不做复杂社区
- 不做复杂商业化系统
- 不做多端铺开

## 6. 推荐 Sprint 切法

为了把重点功能拆得更细，MVP 不再只分两段，而是按下面 5 个 sprint 推进：

1. Sprint 0：信息底座与治理基础
2. Sprint 1：资讯接入与信号准备
3. Sprint 2：潜在题材催化结构化挖掘核心
4. Sprint 3：关联关系与用户排序
5. Sprint 4：输出、复盘与研究闭环

其中：

- Sprint 0 是底座
- **Sprint 2 是第一优先的业务价值 Sprint**

## 7. 当前版本结论

如果只能优先把一件事做深，就优先做：

**潜在题材催化的结构化挖掘结果**

后续所有页面、排序、复盘和预警，都应该围绕这个核心结果来服务，而不是反过来。
