# 第一步：远景目标与核心方向定版（全项目不可动摇的底层锚点）

我们先把**顶层不变的底层逻辑、终局目标、核心边界**彻底定死，后续所有的规则设计、产品开发、运营动作，全部围绕这个锚点展开，绝不跑偏。

## 一、顶层核心：使命与远景终局

### 1. 核心使命（我们为什么要做这件事，解决什么根本问题）

**用AI Agent重构个人投资者的投研全流程，让每一位A股个人投资者，都能拥有专属、合规、高效的自动化投研助理，打破机构与个人之间的投研效率壁垒，把投资者从80%的重复、机械、低效的信息筛选、分析、复盘工作中彻底解放出来**。

### 2. 远景愿景（3-5年，我们最终要做成什么样）

**成为国内AI Agent个人投研赛道的头部基础设施平台，从个人专属投研系统演进为开放的投研 Skill、Agent 与 Workflow 生态，重构个人投资者获取、理解、沉淀和复用市场信息的方式。彻底替代韭研公社、财联社等传统资讯聚合产品的个人用户市场**。

## 二、一句话定死核心业务定位（所有动作的唯一锚点）

**国内首个「合规驱动、Agent原生」的A股个人投资者专属投研全链路自动化闭环工具**。

这句话里的每一个关键词，都是我们和所有竞品的本质区别，也是绝对不能偏离的核心：

1. **合规驱动**：把合规刻进产品底层，不是事后加免责声明，而是从规则设计、Agent开发、内容输出全流程锁死合规边界，这是我们的生死线，也是长期经营的核心护城河；
2. **Agent原生**：产品的底层逻辑是多Agent自动化协同，不是传统工具加个AI插件，从诞生起就是为了实现投研全流程的自动化，而非信息聚合；
3. **A股个人投资者**：核心客群精准锁定，先不碰机构业务，死磕个人投资者（职业股民、资深散户、中小私募、财经内容创作者）的真实痛点，不做全人群覆盖；
4. **专属**：所有能力围绕用户的持仓、关注标的、自定义规则展开，千人千面，一人一模型，彻底告别传统产品「千人一面」的痛点；
5. **投研全链路自动化闭环**：我们做的不是单点工具，不是资讯搬运工，而是覆盖「7×24h信息采集→事件分类→客观归因→风险排查→内容生成→复盘归档」的完整投研流程，给用户的是可直接用于决策的客观结果，而非需要二次加工的零散信息。

## 三、绝对不可逾越的边界红线（我们绝对不做什么）

和「要做什么」同等重要的，是明确「绝对不做什么」，从根源上避免需求漂移、合规踩坑、精力分散：

1. **绝对不碰监管红线**：永远不提供任何证券投资建议、个股推荐、涨跌预测、买卖操作指引，永远坚守「公开信息客观处理工具」的定位，绝不越雷池一步；
2. **绝对不做非核心的冗余功能**：初期绝不碰行情交易、股票社区、UGC内容平台、直播投教等功能，不和同花顺、东方财富、韭研公社拼大而全，只死磕「投研全链路自动化」这一个核心点；
3. **绝对不做机构业务的定制化服务**：前期完全聚焦个人投资者的标准化需求，不做机构客户的定制化开发，避免分散精力，先把标准化产品做透；
4. **绝对不碰版权与数据合规风险**：所有数据源优先采用官方授权API，绝不商用爬取有版权的资讯、研报内容，从源头规避知识产权风险。

## 四、三大核心战略方向（远景拆解的长期深耕主线）

我们所有的产品迭代、技术开发、商业布局，都只围绕这三个核心方向展开，不做任何偏离主线的动作：

### 方向1：把个人投资者围绕“事件信息获取、事件理解、事件跟踪、复盘沉淀”的关键流程，从手工完成重构为 Agent 自动化完成，让用户只关注最终值得看的事件与研究结论。**核心目标不是简单提效，而是在提效的同时提高事件识别质量、降低信息遗漏率、增强复盘连续性。**

这是我们的核心立身之本，也是和所有传统产品的代差级优势。

- **终局目标**：把个人投资者的投研工作，从「用户主动找信息、手动筛事件、自己串逻辑、熬夜做复盘」，彻底重构为「用户简单设置关注范围，Agent 全自动完成全链路处理，只给用户交付客观、精准、和他强相关的决策依据与复盘沉淀」，让用户的日均投研耗时从 3 小时，压缩到 10 分钟以内。

  **分阶段落地路径 & 核心动作**：

  1. 第一阶段（0-6 个月，MVP 期）

     ：死磕「高频复盘用户的每日事件跟踪→事件归一→逻辑梳理→复盘生成」核心链路，把这条最高频的工作流 100% 自动化。

     - 核心动作：拆解短线 / 题材用户的真实复盘流程，固化成标准化的 Agent 执行节点；打磨开箱即用的预设事件过滤规则、事件分类逻辑、复盘生成模板，用户只需设置观察池 / 关注题材，就能一键生成每日复盘，零配置门槛。
     - 可量化里程碑：用户完成一次完整的每日复盘，从原来的手动 2 小时，压缩到系统自动化 5 分钟以内；核心链路自动化执行成功率≥99%。

     

  2. 中期（6-18 个月，产品化期）

     ：延伸覆盖全投研流程，从每日复盘，拓展到题材全周期跟踪、个股基本面跟踪、交易复盘沉淀、自定义规则自动化等场景，完善 Agent 的容错、重试、异常兜底机制，让系统适配更多用户的投研习惯。

     - 核心动作：沉淀标准化的投研 Workflow 模板，优化 Agent 的长链路跟踪能力、多事件关联分析能力，开放轻量化的自定义规则配置。
     - 可量化里程碑：覆盖个人投资者 80% 的投研场景，用户核心需求的自动化满足率≥90%。

     

  3. **终局期（18 个月以上）**：实现全场景、全周期的投研流程自动化，用户只需定义自己的投研框架，Agent 就能完成从信息采集、分析跟踪、复盘沉淀到认知迭代的全流程服务。

  **长期核心优化方向**：持续打磨 Agent 链路的稳定性、事件识别的准确性、逻辑拆解的精准度，不断降低用户的使用门槛，让自动化的效果远超用户手动处理的效果。

  

- 核心动作：把投研的每一个环节，都拆解成可标准化、可自动化、可容错的Agent任务，不断优化链路的稳定性、准确性、效率，让用户的投研耗时从日均3小时，压缩到10分钟以内。

### 方向2：把「合规」做成核心竞争力，而非最低底线

在监管趋严的A股市场，合规是用户最核心的安全感来源，也是我们和竞品最大的差异化优势。

### 合规不只是内容审核

而是 3 层：

#### 1. 输出边界合规

不能碰：

- 个股推荐
- 买卖建议
- 个性化投资建议
- 收益承诺
- 盘中操作指令

#### 2. 流程合规

每个 Agent 不能越权：

- 哪些能总结
- 哪些能归因
- 哪些只能提示风险
- 哪些不能生成

#### 3. 审计合规

所有重要结论必须：

- 有来源
- 有证据
- 有时间线
- 有日志
- 可回看

- **终局目标**：成为国内个人投研工具里，合规体系最完善、监管风险最低、用户信任度最高的产品，让监管放心、让用户安心，在行业合规收紧的周期里，成为用户最安全、最可靠的投研工具选择。

  **核心底层原则**：严格遵守我们定死的「系统可信度三原则」—— 所有输出**有来源、有证据、有时间线**，从根源杜绝 AI 幻觉、主观引导、违规表述。

  **分阶段落地路径 & 核心动作**：

  1. 第一阶段（0-6 个月，MVP 期）

     ：把合规规则嵌入产品底层，从根源锁死合规边界，而非事后加免责声明。

     - 核心动作：

       - 事前拦截：固化全链路敏感词库、违规表述拦截规则，所有 Agent 的输出必须先经过合规校验，违规内容直接拦截，无法流到用户端；
       - 事中约束：所有 Agent 的 Prompt、输出模板强制锁定合规框架，禁止大模型自由生成带主观判断、投资引导的内容；
       - 事后可审计：全链路执行日志、内容生成记录、信息来源完整留存至少 6 个月，做到可追溯、可核查、可举证；
       - 强制合规声明：所有输出内容必须附带标准化合规声明，明确产品的工具定位，划清权责边界。

       

     - 可量化里程碑：系统输出内容 100% 符合合规红线要求，零违规表述、零引导性内容；所有核心事实 100% 标注可溯源的官方来源。

     

  2. 中期（6-18 个月，产品化期）

     ：完善全场景合规体系，把合规从 “底线要求” 变成 “用户可感知的价值”。

     - 核心动作：持续迭代合规校验规则，适配监管政策的变化；针对用户自定义规则、自定义模板，建立前置的合规审核机制；强化信息溯源能力，让用户一键就能查看事件的原始来源、原文证据，提升用户信任度。
     - 可量化里程碑：形成一套可复用、可迭代的金融投研 AI 合规体系，用户调研中，“合规、客观、可信” 成为用户选择我们的 Top3 核心理由。

     

  3. **终局期（18 个月以上）**：建立行业领先的投研 Agent 合规标准，把合规体系变成我们的核心壁垒，甚至可以对外输出合规解决方案，形成额外的品牌与商业价值。

# 方向 5：冷启动工作流闭环

你现在讲了大方向，但还缺一个非常现实的问题：

## 第一批用户为什么今天就要用你？

所以必须新增一条：

## **方向 5：围绕高频场景打透一个最小闭环**

这条不是终局，是打法。

### 你第一阶段的真实产品，不是“平台”

而是一个具体闭环：

**资讯进入 → 事件抽取 → 事件归一 → 重要性排序 → 个股/板块复盘输出**

这个闭环如果没打透，前面 1、2、3、4 都会变成口号。

### 建议表述

**第一阶段聚焦一个高频刚需闭环：围绕用户观察池与关注题材，实现从资讯过滤到事件复盘的全链路自动化，先让用户在一个日常高频动作上形成依赖。**



**核心目标**：放弃大而全的功能规划，死磕一个用户每天都要用、痛点最强的最小闭环，快速验证用户价值、跑通付费逻辑，拿到第一批种子用户和付费用户。

**核心动作**：

1. 钉死最小闭环：「用户设置观察池 / 关注题材→Agent 每日 7×24h 自动过滤噪音→提取核心事件→归一关联溯源→生成标准化每日复盘报告」；
2. 所有开发、优化、测试，都围绕这个最小闭环展开，非闭环内的功能，第一阶段一律不做；
3. 快速上线 MVP，通过种子用户反馈持续优化闭环体验，验证用户的付费意愿，跑通商业闭环。

**可量化成功标准**：

1. 核心用户形成稳定的高频使用习惯，日均打开率≥80%，人均单日使用时长≥5 分钟；
2. MVP 上线 3 个月内，种子用户付费转化率≥10%；
3. 付费用户的月留存率≥60%，用户核心好评点集中在 “省时间、抓重点、复盘方便”。



## 三、系统可信度底层三原则（全链路必须遵守）

这是产品和市面上所有 AI 胡说型产品、资讯搬运产品拉开差距的核心，也是合规与用户信任的底层保障，所有 Agent、所有输出内容必须严格遵守：

1. **有来源**：所有事件、数据、事实，必须标注官方可溯源的信息来源，无来源的内容绝对不输出；
2. **有证据**：所有核心判断、事件拆解，必须对应原文片段、原始公告 / 新闻的核心内容，不做无依据的主观推演；
3. **有时间线**：所有事件必须可追溯演化过程，形成完整的事件时间线，方便用户回溯、复盘、跟踪。

## 四、第一阶段成功验收标准（0-6 个月，做成了的明确标尺）

避免产品空转、功能自嗨，明确第一阶段的核心验收指标，所有开发、优化都围绕这些指标展开：

### 1. 产品体验指标

- 用户完成冷启动（设置观察池 / 关注标的）的时长≤1 分钟；
- 用户打开产品后，能在 3 分钟内找到「今天最值得关注的核心事件」；
- 核心链路（事件采集→过滤→生成复盘）的自动化执行成功率≥99%。

### 2. 用户价值指标

- 核心用户日均打开频次≥1 次，日均使用时长≥5 分钟；
- 用户调研中，90% 以上的用户认为「系统比自己手动刷资讯更省时间」；
- 80% 以上的用户，能通过系统完成至少 1 次完整的事件跟踪与深度复盘。

### 3. 系统能力指标

- 事件归一、去重的准确率≥95%，重复 / 无效信息过滤率≥90%；
- 核心事件的识别延迟，不晚于主流资讯平台的发布时间；
- 所有输出内容 100% 符合合规红线要求，无违规表述、无敏感词。

### 4. 商业验证指标

- 种子用户付费转化率≥10%；
- 付费用户的月留存率≥60%；
- 用户明确认可「愿意为这套事件认知与复盘能力付费」，验证产品市场匹配度（PMF）。
