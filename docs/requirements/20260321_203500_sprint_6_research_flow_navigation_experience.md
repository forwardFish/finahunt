# Sprint 6 需求文档

## 名称

Finahunt 页面分类重构与完整 UI/browser 验收

## 背景

当前 `finahunt` 首页把“今日入口、主线发酵、低位研究、工作台总览”堆在同一页里，导致：

- 顶层导航名义上有分类，实际仍停留在同一页
- 用户无法通过 URL 明确进入某一类研究视图
- 页面信息密度过高，难以判断“今天先看什么”
- 旧链接 `/sprint-2` 和 `/low-position` 与真实页面职责不再匹配

## 产品目标

把前端按研究流程拆成 4 个独立页面：

- `/` 今日入口
- `/fermentation` 主线发酵
- `/research` 低位研究
- `/workbench` 工作台总览

并保留兼容入口：

- `/sprint-2 -> /workbench`
- `/low-position -> /research`

## 页面职责

### 1. 今日入口 `/`

只负责“今天先看什么”，保留：

- 今日概览
- 主线摘要
- 低位研究摘要
- 关键事件入口
- 风险边界
- 去三个下游页面的 CTA

不再承载：

- 全量矩阵
- 全量候选公司列表
- 完整证据带

### 2. 主线发酵 `/fermentation`

只负责“什么题材在发酵”，保留：

- 发酵阶段分布
- 主题卡
- 热度 / 催化 / 持续性
- 关键证据
- 去工作台深挖入口

### 3. 低位研究 `/research`

只负责“哪些低位机会值得深挖”，保留：

- 低位题材卡
- 候选公司
- 研究理由
- 验证状态
- 风险提示

### 4. 工作台总览 `/workbench`

只负责“完整聚合视图”，保留：

- 主线与低位双栏总览
- 全景矩阵
- 证据带
- 代表性消息卡

## 路由契约

- 顶层导航必须跳到独立 URL
- 不允许继续用页内 tab 伪装成分类拆页
- 兼容路由必须可访问并跳转到新页面

## 数据约束

- 优先复用 `dailySnapshot.ts`
- 优先复用 `lowPositionWorkbench.ts`
- 不重写 runtime artifact contract
- 页面只是重组已有数据，不改变 runtime 产物含义

## 验收要求

- `/`、`/fermentation`、`/research`、`/workbench` 全部返回 `200`
- `/sprint-2` 能跳转到 `/workbench`
- `/low-position` 能跳转到 `/research`
- 首页不再出现完整工作台堆叠
- 主线发酵页只聚焦主线
- 低位研究页只聚焦低位研究
- 工作台总览页承接完整聚合视图
- 必须进行 browser 级验证，确认导航点击后进入独立 URL
