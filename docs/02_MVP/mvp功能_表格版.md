# Finahunt MVP 功能表格版

## 当前最高优先级

当前最高优先级不是“做更多资讯展示”，而是先把：

**潜在题材催化的结构化挖掘结果**

做深做准做可追溯。

## 产品一句话

Finahunt 的 MVP 要先验证一件事：把合规金融资讯压缩成用户可消费的事件流、题材线索和日终复盘，让用户不再手工刷大量信息源。

## 范围原则

- 只做资讯认知闭环，不做交易闭环。
- 只做合规公开信息，不接私有群和灰色数据。
- 只做研究辅助，不做荐股、收益承诺和自动交易。

## P0 功能验收表

| 功能项 | 最低验收标准 | 主要输出 |
| :--- | :--- | :--- |
| 合规资讯源接入 | 至少接入 3 类公开资讯源；每条资讯都带来源和时间 | 原始资讯记录 |
| 清洗去重 | 同源重复内容可折叠；明显无关内容可过滤 | 清洗后资讯流 |
| 事件抽取 | 每条事件都有标题、时间、来源、证据片段 | 事件对象 |
| 事件归一 | 同一事件多源报道合并为单条事件流 | 事件主线 |
| 题材识别 | 每条事件可映射到主题；映射有证据 | 题材标签 |
| 催化识别 | 至少区分政策、业绩、产业、资金、舆情等催化类型 | 催化标签与级别 |
| 个股/板块关联 | 事件和个股、板块关系可解释 | 关系链路 |
| 用户相关度排序 | 同一批事件能按用户观察池和题材偏好重排 | 今日重点列表 |
| 今日重点页 | 3 分钟内看完当日重点 | 页面或等价输出 |
| 观察池事件页 | 能围绕用户观察池看事件流变化 | 页面或等价输出 |
| 日终复盘页 | 自动生成结构化复盘，不含荐股表达 | 日报或等价输出 |
| 合规治理 | 所有输出保留来源，带风险提示，不含高风险表述 | 合规输出 |
| 日志与人工校正 | 失败可追踪，关键字段可人工修正 | 审计日志与校正记录 |

## P1 功能验收表

| 功能项 | 目标 | 说明 |
| :--- | :--- | :--- |
| 异动预警中心 | 让用户知道什么值得立刻看 | 先做站内提醒，不做复杂通知系统 |
| 历史相似案例基础版 | 让用户看到过去类似事件的演化 | 只做参考，不做预测 |
| 个人研究归档 | 让研究内容可回看、可沉淀 | 先做个人维度，不做分享 |
| 个股/题材详情增强 | 让关系链更可解释 | 先强化详情，不做复杂图谱系统 |
| 轻量提醒 | 提升留存和回访 | 先做轻提醒，不做强运营 |

## P2 暂缓表

| 模块 | 暂缓原因 |
| :--- | :--- |
| 社区和 UGC | 容易分散 MVP 的核心价值验证 |
| 复杂个性化能力 | 实现成本高，容易把规则做重 |
| 商业化系统 | 应该建立在核心留存和价值验证之后 |
| 多端形态 | 先把 Web/单端闭环做好 |
| 荐股 / 预测 / 自动交易 | 合规和风险都不适合当前阶段 |

## 推荐开发顺序

1. 先完成底座、资讯接入和标准化。
2. 再优先完成潜在题材催化的结构化挖掘结果。
3. 然后完成个股/板块关联和用户相关度排序。
4. 最后补三类核心输出页和日终复盘。
