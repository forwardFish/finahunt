# 操作手册

## 本地初始化
1. 创建 Python 3.11+ 环境。
2. 执行 `pip install -e .[dev]`。
3. 执行 `pytest`，确认基础测试通过。

## 运行最小 runtime 流程
1. 执行 `python -c "from workflows.runtime_schedule import run_runtime_cycle; print(run_runtime_cycle())"`。
2. 检查 `results.source_runtime` 是否包含 `raw_documents` 和 `registry_snapshot`。
3. 检查 `results.compliance_guard` 是否包含 `allowed_documents`、`blocked_documents` 和 `manual_review_required`。
4. 检查 `results.normalize` 是否包含 `normalized_documents` 和 `format_validation_report`。
5. 检查 `results.source_audit` 是否包含 `trace_report` 和 `runtime_audit_log`。

## 人工校正介入时机
出现以下任一情况时，必须进入人工校正：
- `blocked_documents` 非空。
- 输出中存在 `high` 风险等级。
- 事件对象缺少 `source_refs` 或 `evidence_refs`。
- final gate 要求 `requires_human_review = true`。

## 人工校正记录要求
每条人工校正记录都必须包含：
- `correction_id`
- `target_stage`
- `target_field`
- `original_value`
- `corrected_value`
- `operator`
- `reason`
- `approved`

## Final Gate 人工签收要求
进入 final gate 前，必须确认：
1. runtime trace 可追踪到 source、compliance、normalize、audit 四个阶段。
2. 所有高风险项都有人工校正记录或明确阻断说明。
3. `final_gate_decision` 为 `PASS` 或 `PASS_WITH_NOTES` 时，必须有证据链和 action recommendation。

## 常用验证命令
- `python tools/gate_check/validate_norms.py`
- `python -m pytest tests/spec/test_traceability_registry.py`
- `python -m pytest tests/spec/test_gate_check.py`
- `python -m pytest tests/integration/test_runtime_foundation.py`
- `python -m pytest tests/e2e/test_final_gate.py`
